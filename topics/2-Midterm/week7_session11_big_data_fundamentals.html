What Is Big Data?

Big Data

Datasets so large, fast-moving, or complex that traditional data processing tools (Excel, basic databases) cannot store, manage, or analyze them effectively. Requires specialized technologies and approaches.

06

Not Just "Lots of Data"

Big Data isn't just about SIZE. It's about data that is too big, too fast, or too varied for normal tools.

Your class attendance sheet? Not Big Data (Excel handles it fine).
All Facebook posts from 3 billion users in real-time? THAT's Big Data.

1. Volume

The AMOUNT of data generated. We're talking petabytes and exabytes, not gigabytes.

SourceData Generated
Facebook4 petabytes/day (4,000 terabytes)
YouTube500 hours video uploaded per minute
Walmart2.5 petabytes customer data/hour
Large Hadron Collider1 petabyte/second during experiments
10

2. Velocity

How FAST data is generated and needs to be processed. Some data is useless if not analyzed in real-time.

Speed LevelExampleWhy Speed Matters
Batch (hours/days)Monthly sales reportHistorical analysis OK
Near real-time (minutes)Social media trendingNeed to react quickly
Real-time (milliseconds)Stock trading, fraud detectionDelay = money lost
11

3. Variety

Data comes in MANY different formats. Not just neat rows and columns.

TypeDescriptionExamples
StructuredOrganized in rows/columnsDatabases, spreadsheets, CSV
UnstructuredNo predefined formatVideos, images, social posts, emails
Semi-structuredSome organization, flexibleJSON, XML, HTML, log files
12

4. Veracity

How TRUSTWORTHY is the data? Dirty data leads to wrong decisions.

  • Incomplete records (missing fields)
  • Inconsistent formats (dates: 07/10/2026 vs July 10, 2026)
  • Duplicates (same customer counted twice)
  • Outdated information (old addresses, dead emails)
  • Fake data (bots, spam, manipulated reviews)
13

5. Value

Data is worthless until you extract INSIGHTS from it.

Having 1 billion customer records means nothing if you can't answer: "Which customers are about to leave?" or "What product should we launch next?"

02 Part 2

Data Types

Structured vs. Unstructured vs. Semi-structured

16

Structured Data

Characteristics

  • Organized in tables (rows & columns)
  • Fixed schema (predefined format)
  • Easy to search and query (SQL)
  • Only ~20% of all data

Examples

  • Student grades database
  • Bank transaction records
  • Employee payroll
  • Inventory spreadsheets
17

Unstructured Data

Characteristics

  • No predefined organization
  • Can't fit in traditional database
  • Hard to search/query directly
  • ~80% of ALL data is unstructured!

Examples

  • Videos and images
  • Social media posts
  • Emails and chat messages
  • Audio recordings, PDFs
18

Semi-Structured Data

Characteristics

  • Has some organization (tags, keys)
  • Doesn't fit strict table format
  • Flexible schema
  • Common in web/apps

Examples

  • JSON (API responses)
  • XML documents
  • HTML web pages
  • Server log files

Hadoop: The Pioneer

Problem: one computer can't process petabytes. Solution: split the work across THOUSANDS of computers working in parallel.

Analogy: You have 1 million books to read. One person? Takes 100 years. 1000 people each reading 1000 books? Done in weeks. Hadoop splits data across many machines the same way.

27

Data Lake vs Data Warehouse

Data Lake

  • Store RAW data (any format)
  • Schema-on-read (decide structure later)
  • Cheap storage
  • For data scientists/explorers
  • Like a real lake: everything flows in

Data Warehouse

  • Store PROCESSED, structured data
  • Schema-on-write (structured before storing)
  • Expensive, optimized for queries
  • For business analysts/reports
  • Like a library: organized and catalogued
