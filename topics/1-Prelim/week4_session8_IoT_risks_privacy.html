07Case

Mirai Botnet (2016)

  1. What: Malware infected 600K+ IoT devices
  2. How: Scanned for default passwords (admin/admin)
  3. Attack: Used devices to DDoS major websites
  4. Impact: Twitter, Netflix, Reddit all DOWN
  5. Lesson: Weak IoT security = global internet risk
08Why Vulnerable

Why IoT Is Easy to Hack

  • Cheap manufacturing: security costs money
  • No updates: can't receive patches
  • Default passwords: users never change
  • Limited computing: can't run complex security
  • Long lifespan: deployed years without updates
  • Users don't think: "it's just a lightbulb"

18The Law

RA 10173: Data Privacy Act

Data Privacy Act protects personal information in all systems, government and private. Applies to ALL data processing including IoT collection.

19Requirements

What RA 10173 Requires

  • Consent: inform and get permission
  • Purpose limitation: use only for stated reason
  • Proportionality: collect only what's needed
  • Transparency: tell people what and why
  • DPO: appoint Data Protection Officer
  • Breach notification: report within 72 hours

Evaluating
IoT Solutions

25Framework

7-Question Evaluation

  1. Problem real? Significant?
  2. Who benefits? Users, company, both?
  3. Data collected? Minimum necessary?
  4. Security? Encryption, updates, access?
  5. What could go wrong? Worst case?
  6. Proportionate? Benefit justifies risk?
  7. User control? Opt-out, deletion, transparency?
26Good Case

Flood Sensors for Barangay

QuestionAnswer
Problem?Flooding causes deaths. No early warning.
Benefits?Residents safety. LGU disaster prep.
Data?Water level only. No personal data.
Security?Low risk (no personal data)
Proportionate?YES. Life-saving, minimal privacy risk.

Verdict: DEPLOY. Low risk, high public benefit.

27Bad Case

Facial Recognition Attendance

QuestionAnswer
Problem?Attendance takes 5 min. Moderate.
Data?Biometric data of MINORS. Sensitive.
Worst case?Breach exposes children permanently
Proportionate?QUESTIONABLE. 5 min vs lifetime risk?
Alternatives?QR code, RFID card (no biometric risk)

Verdict: DISPROPORTIONATE. Simpler alternatives exist with far less risk.

29Part 5

Best Practices

30Personal

IoT Security Tips

  1. Change default passwords immediately
  2. Update firmware when available
  3. Separate WiFi for IoT devices
  4. Disable unused features (mic, camera)
  5. Research before buying: updates available?
  6. Review privacy settings
31Organizations

Org Security Checklist

  • Inventory all IoT devices on network
  • Segment IoT from main systems
  • Encrypt data in transit
  • Access control who manages devices
  • Regular audits for vulnerabilities
  • Incident plan for breaches
  • Vendor assessment security track record
32Rule

Good innovation must be:
Useful + Safe + Socially Acceptable

Fails any one? Redesign.

33BSIS Role

Your Responsibility

  • Design systems that protect users (privacy by design)
  • Collect only what's needed (minimization)
  • Build security IN, not afterthought
  • Advocate for users when leadership pushes invasive
  • Stay updated on law and security
  • "Can" ≠ "should"
34Summary

Key Takeaways

  • IoT vulnerable: weak passwords, no updates, unencrypted
  • Privacy: who owns, accesses, stores data
  • RA 10173 protects Filipinos (including IoT data)
  • 7-question framework evaluates any IoT solution
  • Proportionality: does benefit justify risk?
  • Good innovation = useful + safe + acceptable
